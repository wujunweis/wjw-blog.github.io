斐切纳波数列

实现方法 简单总结 for/while、递归

  • 斐波那契数列，是指这样的一组数列： 1、1、2、3、5、8、13、21……在数学上，斐波那契数列被以递归的方法定义：F0=0，F1=1，Fn=Fn-1+Fn-2（n>2, n∈N*），用文字说，就是斐波那契数列由0和1开始，之后的斐波那契数列系数就由之前的两个数相加之和。

  • 常用的计算方法由两大类：递归/循环 ，后面会补充所有算法使用

  • 计算的时候可以加入console.time('id') 与 console.timeEnd('id') 如果你还不知道console的用发可以点击这里

递归

常见递归

  • 优点：代码简洁清晰。

  • 缺点：存在重复计算，浪费内存（递归是一个堆栈计算过程，数字过大会超出内存不够的情况）

  • example： n=5 -> fib(4)+fib(3),fib(3)+fib(2) 大于50可能浏览器就出现假死现象

    console.time('runTime');
function fib(n)
{
    if(n === 1 || n === 2) return 1;
    return fib(n - 2) + fib(n - 1);
}
console.log(fib(30));//832040
console.timeEnd('runTime');//runTime: 9.702880859375 ms

优化递归：思路将前两个数作为参数避免这里的重复计算

  • 优点：1 .可以说省去大量重复计算，相当的快了（因为只有对应需要的才会加） 2. 代码简洁，易懂

    console.time('runTime');
function fib(n){
   function f(n, a, b) {
       if(n === 1){
           return a;
       }
       if(n === 2){
           return b;
       }
       else
       {
           return f(n - 1, b, a + b);
       }
   }
   return f(n, 1, 1);
}
console.log(fib(30));//832040
console.timeEnd('runTime');//runTime: 0.105712890625 ms

优化递归：既然可以把数字作为参数传递，那么把运算结果存储在数组内使用闭包，也能避免计算（闭包，数组引用传值）

  • 优点： 利用闭包+数组计算避免，数据重复计算

  • 缺点： 需要操作数组赋值会有相应的消耗，代码可读性不是很友好（比上面的纯数字要慢）

    console.time('runTime');
var fib = function (){
    let memo = [0, 1];
    let f = function(n) {
        if (memo[n] === undefined) {
            memo[n] = f(n - 2) + f(n -1);
        }
        return memo[n];
    }
    // 因外层为立即执行函数 真正使用的其实是本身 ‘f’ 函数
    return f;
}()
console.log(fib(30));//832040
console.timeEnd('runTime');//runTime: 0.169921875

优化递归：既然闭包能存储计算结果那么，将计算结果直接提取成计算函数呢？

  • 优点：将闭包拆分出来，方便阅读理解，让函数更加纯粹

  • 缺点：增加了逻辑判断与执行行，可能会产生一定的消耗，整体上处理时间与匿名闭包差不多

    console.time('runTime');
var memoizer = function(func){
    let memo = [];
    return function(n){
        if(memo[n] === undefined) {
            memo[n] = func(n);
        }
        return memo[n];
    }
}

var fib = memoizer(function(n){
    if(n === 1 || n === 2) {
        return 1;
    }
    return fib(n - 2) + fib(n -1);
});
console.log(fib(30)); //832040
console.timeEnd('runTime');//runTime: 0.2080078125 ms

循环

普通for 循环【迭代算法】

  • 优点：代码简洁易懂，纯粹的计算数值

  • 缺点：存在多变量，需要计算赋值

    console.time('runTime');
function fib(n) {
   let a = 1, b = 1, sum;
   for(let i = 2; i < n;i++) {
       sum = a + b;
       a = b;
       b = sum;
   }
   return sum;
}
console.log(fib(30));//832040
console.timeEnd('runTime');//runTime: 0.130126953125 ms

for 循环【迭代算法】： 怎么改进 思路去掉sum变量，我们其实只需要第二个值即第n个的a与b的加和。比如 【a, b】 我们实际得到要的值是 b 个的【a + b】 然后去实现试试

  • 优点：运用解构赋值，代码简洁，省去了不必要的变量

  • 缺点：利用数组解构，增加了复杂难度会有一定的时间消耗

    console.time('runTime');
let fib = function(n) {
   let a = 1, b = 1;
   for(let i = 2; i < n;i++) {
       [a, b] = [b, a + b];
   }
   return b;
}
console.log(fib(30));//832040
console.timeEnd('runTime');//runTime: 0.14599609375 ms

while 逆向思维【迭代算法】

  • 优点：代码简洁，省去了计算i值，比递归计算快

  • 缺点：不易理解，取的值 b即为sum值

    console.time('runTime');
let fib = function(n) {
   let a = 0, b = 1;
   while(n--){
       b = a + b;
       a = b - a;
   }
   return b;
}
console.log(fib(30));//832040
console.timeEnd('runTime');//runTime: 0.178955078125 ms
  • 正常来说单层for 循环运算应该是最快的>优化过的递归操作>普通递归 （这里从控制台输出的为什么不是这个结果呢？其实跟当前计算的数值大小有关，当计算数值大于50 或 100 以上的时候这个时间差会越来越明显。
    也可以理解为层数偏少的计算递归也有一定的优势 example： 普通for 300 就比 递归快 将近0.1 ms
    合理的选择与自己产品相应的计算方法才是最好的～
